# Plan: Fix Mobile Keyboard Delay + Add /five Wordle Route

## Objective

Two independent improvements:
1. Fix ~500ms input delay on mobile when pressing keyboard keys
2. Add a 5-letter daily Wordle clone at `/five` (today: "dusty", #1795)

## Task 1: Fix Mobile Keyboard Input Delay

**Root cause:** Two sources of delay on mobile:
1. `.keyboard` container inside `overflow-y: auto` causes browsers to wait briefly before deciding if a touch is a scroll or tap — even if individual `.key` buttons have `touch-action: manipulation`. Adding it to the container eliminates the delay for the entire keyboard area.
2. Zustand `persist` middleware writes to `localStorage` synchronously on every state update (every letter press). On mobile, this can block the main thread.

**Changes planned:**
- `src/styles/tiles.css`: Add `touch-action: manipulation` to `.keyboard`
- `src/hooks/useGame.ts`: Replace direct `createJSONStorage(() => localStorage)` with a debounced adapter that delays writes 300ms

## Task 2: Add /five Route (5-letter Wordle)

**What:** A Wordle clone at `/five`. Today's answer is "dusty", puzzle #1795. No navigation buttons to it — user types URL directly. User updates `fiveConfig.ts` daily.

**Files planned:**
- `src/data/fiveWords.ts`: Auto-generated from `scripts/raw/targets.json` + `dictionary.json`, filtered to 5-letter words
- `src/data/fiveConfig.ts`: Manual daily config (FIVE_ANSWER, FIVE_PUZZLE_NUMBER)
- `src/components/Row.tsx`: Add `wordLength` prop (default 6)
- `src/components/Board.tsx`: Add `wordLength` prop (default 6), pass to Row
- `src/hooks/useFiveGame.ts`: Zustand store for 5-letter game, no persist, fixed answer from config
- `src/components/FiveGame.tsx`: UI component with simple inline end-game modal
- `src/main.tsx`: Add `/five` pathname check → render FiveGame

**Constraints:**
- No hard mode validation
- No navigation buttons from main app
- `scoreTiles` works for any word length — reuse unchanged
- Same keyboard component (26 letters) works fine for 5-letter game
- In-memory only (no localStorage persistence)
